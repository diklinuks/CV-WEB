Betsy

Autonomous procurement agent for a small manufacturer. Single-agent ReAct system that monitors inventory, evaluates suppliers, generates purchase orders, and learns from outcomes over a 90-day simulation.

  • LangGraph
  • MCP tools
  • Gemini 2.5 Flash
  • PostgreSQL · pgvector
  • FastAPI
  • Python 3.12
  • Docker
Code on GitHub ↗

What Betsy does

Betsy is a single autonomous agent that runs the procurement function for a small manufacturer. Each simulated day she watches inventory levels, decides when something needs reordering, weighs suppliers on price, lead-time, and reliability, places purchase orders, tracks deliveries, reconciles invoices, and reflects on the outcome of each decision. High-value or unusual decisions pause for human approval rather than executing autonomously.

Architecture

The agent is a LangGraph create_react_agent with tools exposed via an MCP (Model Context Protocol) server. LangGraph consumes the tools through langchain-mcp-adapters, which means the same tool definitions can be used from the agent or independently. The LLM (Gemini 2.5 Flash) proposes — the actual decisions, caps, and writes are enforced in code, not by the model.

The split between LLM and code

This is the principle that makes the system trustworthy:

  • LLM proposes — suggests which supplier, how much, when.
  • Code decides and enforces — applies caps, checks blocked-supplier lists, writes to the database, ensures the audit log is append-only.
  • LLM reflects — after an outcome, it writes a one-paragraph lesson which gets embedded into the vector store.

Memory

Two layers: structured data (suppliers, orders, deliveries, invoices) lives in PostgreSQL. Rationale and learned lessons live in pgvector. When a similar reorder situation comes up, Betsy retrieves relevant past lessons before deciding, so she doesn't repeat mistakes.

The Jenny dashboard

A FastAPI web app where the human operator (Jenny) runs the simulation, watches it tick, approves or rejects flagged decisions, inspects the audit log, tunes the scoring weights, and reads the final report. Rejections require a reason — the reason becomes a learned lesson.

Ethics & safety

Hard guardrails enforced in code, not by the LLM: spending caps, blocked-supplier lists, append-only audit log, and privacy projections on what the agent sees. The agent cannot bypass these regardless of what it proposes.

Observability

LangGraph Studio provides a visual, step-through view of the graph. LangSmith tracing captures every LLM call. The MCP server can be inspected on its own. If MCP setup is awkward, a single env var (BETSY_TOOLS_TRANSPORT=local) wires the identical tools in-process.

How to run it

Full setup is in the repo README. The short version: 

docker compose up -d
python3.12 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env  # paste your GEMINI_API_KEY
python -m app.main load
uvicorn app.web.main:app --reload --port 8000

Without a Gemini key the simulation still runs using a transparent rule-based fallback, so you can see the system without spending anything.

Tests

pytest tests/ -q covers the pure logic — supplier scoring and scenario scheduling — without spinning up the LLM or database.
